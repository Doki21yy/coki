Building Is the Easy Part Now | Mike Krieger on What AI Changed
Instagram 联合创始人、现任 Anthropic Labs 产品负责人 Mike Krieger 在 AI & I 播客中分享了他对 2026 年产品开发范式的深度反思。 核心结论：AI 让「建造」变得前所未有的简单，但「决定建造什么」和「保持产品直觉」反而成了最稀缺的能力。 【建造已 trivial，但剪枝仍是艺术】Krieger 指出，今天的模型不仅能帮你从 0 到 1，甚至能从 0 直接到终态——几小时内完成。他让 Claude 用两小时重建了当年失败的产品 Bourbon，包含了 Bourbon 本身都没有的滤镜功能。但问题在于：模型擅长做加法，不擅长做减法。「产品设计中的艺术与科学，在于知道该放什么进去，而这些直觉需要时间养成。」 【室内树问题】Krieger 提出了一个精妙的「室内树」比喻：传统开发是让树在风中慢慢成长，每一步都暴露在真实用户反馈中；而 AI 加速后的开发更像是把整棵树在室内一次性培育好再拿出来——缺少了每个阶段与真实环境碰撞所形成的韧性。他以自己做的产品 Proof 为例：vibe coding 太上头，一口气加了太多功能，最后推翻重来，借鉴了 Monologue（极简语音笔记 app）的思路重新设计，才实现病毒式传播。 【更愿意重写】面对加速的开发节奏，Krieger 的团队学会了更果断地重写。传统观点认为 Fred Brooks 的「不要重写软件」是铁律，但现在有了模型的 diff 能力加持，重写不再是年度工程，而是几天内的事。他们在 pre-launch 阶段经常建完 V1 发现过度复杂化，直接拆掉建 V2。 【Agent Native 是 2025 新标杆】Krieger 盛赞 Claude Code 是「2025 年最纯正的 agent native 产品」——它的每个 primitive 都能被 Claude 操作和理解。内部甚至把 agent native 包装成了 skill，让 Claude 在原型设计时自动以 agent native 思维工作。Anthropic 内部也在思考：如何让 Claude 建造的软件天生就更具 agent native 特性，而不只是给传统软件打补丁。 【Hiring 的双向变化】Krieger 观察到 hiring 正在双向分化：一方面需要高级系统架构师（分布式系统经验依然稀缺），另一方面需要极度聚焦、有强烈 conviction 的 founder 型人。他说最近招了一位 GM，技术背景轻但产品直觉极强——放在一年前根本不可能，因为当时的 coding 模型还不够好。「当一个 feature 成长到能独立成为产品时，它就需要自己的 team 了。AI 让一个人能 hold 的 scope 变大了，但那条线依然存在。」 【信任的转移】谈及 OpenClaw 时，Krieger 分享了一个有趣观察：OpenClaw 让用户创造了「R2C2」（Doki 的）和「Shelly」（他女朋友的），产生了强烈的归属感。他还提到了「影子 org chart」概念：当组织里每个人都开始用各自的 agent，每个 agent 就会变成那个人的专长代理，最终形成一种新的工作结构。 这对从业者意味着什么： • 建造门槛已大幅降低，聚焦和剪枝能力比编码能力更稀缺 • 愿意快速重写、快速 pivot 是团队的核心竞争力 • Agent native 不只是功能，而是一种设计哲学——让软件的每个 primitive 都能被 AI 操作
